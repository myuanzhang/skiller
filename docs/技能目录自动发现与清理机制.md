# 技能目录自动发现与清理机制

> 本文说明技能中心如何自动发现手动放入中央仓库的 Skill、以及如何自动清理已被删除的 Skill 记录，保证数据库与磁盘状态一致。

## 背景

技能中心（中央仓库 `~/.skiller/skills/`）的所有 Skill 都在 SQLite 数据库（`~/.skiller/skiller.db`）中有对应记录。但存在两种不一致场景：

1. **孤儿技能（Orphan Skill）**：用户手动将 Skill 文件夹（含 `SKILL.md`）放入 `~/.skiller/skills/`，但未通过应用导入流程，数据库中无记录 → 技能中心不显示。
2. **缺失技能（Missing Skill）**：用户手动删除了 `~/.skiller/skills/` 下的 Skill 文件夹，但数据库中仍有记录 → 技能中心显示该 Skill，点击详情报错"没有找到文档文件"。

本机制通过启动时扫描 + 运行时文件监视，双向同步数据库与磁盘状态。

## 两个核心操作

### 1. 孤儿技能发现（discover_orphan_skills）

扫描中央仓库目录 `~/.skiller/skills/`：

- 跳过 `.` 开头的隐藏目录（如 `.skiller/` 元数据目录）
- 跳过数据库中已有记录的目录（通过 `central_path` 比对）
- 只认包含 `SKILL.md` 或 `skill.md` 的目录（由 `skill_metadata::is_valid_skill_dir()` 判断）
- 解析 frontmatter 获取 `name` 和 `description`，fallback 使用目录名
- 自动注册为 `source_type: "local"`、`update_status: "local_only"`、`enabled: true` 的技能记录
- 新发现技能时同步更新 metadata 文件

### 2. 缺失技能清理（cleanup_missing_skills）

遍历数据库中所有技能记录：

- 检查 `central_path` 指向的目录是否仍存在
- 若目录已删除：清理该技能部署到各 Agent 工具目录的 symlink（通过 `sync_engine::remove_target`），然后删除数据库记录
- 清理多个技能后同步更新 metadata 文件

## 触发时机

### 启动时

两个操作在 `initialize_store_inner()` 中按顺序执行：

1. `reindex_from_metadata` — 从 metadata 文件重建索引
2. **`cleanup_missing_skills`** — 清理已删除的技能
3. **`discover_orphan_skills`** — 发现新加入的技能
4. `restore_all_skills_sync_included` — 恢复同步包含状态
5. `ensure_default_startup_scenario` — 应用启动场景

启动日志中会输出 `cleanup_missing_skills removed N skills` 和 `discover_orphan_skills registered N skills`。

### 运行时（文件监视器）

文件监视器（`file_watcher.rs`，基于 `notify` crate）递归监视中央仓库目录和各 Agent 技能目录。当检测到文件系统变化时：

- 经过 500ms 防抖后，在向前端发送 `app-files-changed` 事件**之前**，执行 `reconcile_skills_with_disk()`，即先 cleanup 再 discover
- 额外的 3 秒节流（`ORPHAN_SCAN_INTERVAL`）避免频繁磁盘 I/O
- 前端收到事件后调用 `refreshAppData()` 重新拉取技能列表，UI 自动更新

触发路径有两个：
1. **FS 事件触发**：用户增删文件/目录时，notify 回调 → 防抖 → reconcile → emit
2. **轮询同步触发**：每 3 秒 `sync_watch_set` 检查是否需要新增/移除监视路径，若监视集合变化也会触发 reconcile

## 设计要点

- **节流而非防抖**：磁盘对账使用 3 秒固定间隔节流，而非跟随 500ms 的 UI 防抖，避免连续 FS 事件导致重复扫描
- **清理先于发现**：先删后增，避免同一个目录名在增删交替时出现 ID 冲突
- **不区分来源类型**：所有 source_type（git / skillssh / local）的技能，只要 `central_path` 不存在就清理。因为所有技能的正本都在中央仓库目录下，目录被删意味着用户主动移除
- **symlink 级联清理**：自动清理该技能在各 Agent 工具目录下部署的 symlink，避免留下悬空链接
- **metadata 同步**：增删操作后自动写入 metadata 文件，保证元数据与 DB 一致
