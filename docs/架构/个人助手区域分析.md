# 个人助手（Personal Agents）区域分析

> **命名变更历史**：该分类最初因 OpenClaw 系列产品（Claw = 钳子/龙虾爪）戏称为"龙虾 Agent"，2026-07-11 发现 Hermes Agent、WorkBuddy 等产品与"龙虾"无关、命名不准确，统一更名为「个人助手 / Personal Agents」。内部枚举从 `ToolCategory::Lobster` 改为 `ToolCategory::Personal`，路由从 `/lobster-workspace` 改为 `/personal-workspace`，并通过 serde alias 和前端路由重定向保持向后兼容。

## 1. 定位：与"全局 Workspace"并列的第二大工具分类

项目中仅有两个顶层工具分类：

| 分类 | 含义 | 路由前缀 | 代表产品 |
|---|---|---|---|
| `coding` | 编程类 Agent（默认） | `/global-workspace` | Claude Code、Cursor、Codex、TRAE IDE、Windsurf、Cline 等（共 30+ 个） |
| `personal` | **个人助手类 Agent** | `/personal-workspace` | OpenClaw、Hermes Agent、QClaw、EasyClaw、AutoClaw、WorkBuddy |

注释原文：`Personal AI assistants (OpenClaw, Hermes Agent, WorkBuddy, etc.)` — 即面向通用个人助手场景的 Agent，区别于专注编程的 coding 分类。

## 2. 架构：完全复用 Coding Workspace 的 UX

两者使用**同一个 `WorkspaceView` 组件**，通过 `WorkspaceConfig` 配置差异，见 `app/src/views/workspaceConfigs.ts`：

- 共享相同的页面结构、skill 列表、拖拽排序、详情面板等所有交互
- 差异仅在：标题文案、空状态文案、URL 路径、分类过滤字段
- Sidebar 也是同一个 `renderToolGroup` 函数渲染两个分组，UX 完全一致（可折叠标题 + "全部"概览入口 + 已安装 Agent 列表 + 拖拽排序）

相关文件：
- `app/src/views/workspaceConfigs.ts` — WorkspaceConfig 定义与两个分类的配置常量
- `app/src/App.tsx` — 路由注册（`/personal-workspace` 和 `/personal-workspace/:agentKey`），含旧 `/lobster-workspace` 路径的重定向
- `app/src/components/Sidebar.tsx` — 侧边栏分组渲染逻辑

## 3. 后端识别逻辑

- **分类是纯 UI 分组**：`ToolCategory` 枚举注释明确说明："Does not affect skill deployment, sync, or any other backend behavior — purely a UI taxonomy"（`app/src-tauri/src/core/tool_adapters.rs:5-6`），不影响 skill 部署/同步等任何后端行为，仅用于侧边栏展示分组。
- 通过每个 `ToolAdapter` 上的 `category: ToolCategory::Personal` 字段标记归属。
- 检测机制与 coding agents 完全相同：检测用户家目录下是否存在对应目录（如 `~/.openclaw`、`~/.hermes` 等），存在则视为"已安装"。
- 前端类型定义在 `app/src/lib/tauri.ts:5`：`export type ToolCategory = "coding" | "personal";`
- **向后兼容**：Rust 枚举通过 `#[serde(alias = "lobster")]` 可反序列化旧设置文件中的 `"lobster"` 值，前端 localStorage 排序 key 从 `skiller:lobster-tool-order` 自动迁移到 `skiller:personal-tool-order`，旧路由 `/lobster-workspace/*` 通过 React Router Navigate 自动重定向。

## 4. 内置的 6 个个人助手 Agent

全部定义在 `app/src-tauri/src/core/tool_adapters.rs`：

| key | 名称 | skills 目录 | 特殊配置 |
|---|---|---|---|
| `openclaw` | OpenClaw | `~/.openclaw/skills` | Claw 生态核心 |
| `hermes` | Hermes Agent | `~/.hermes/skills` | **recursive_scan: true**（递归扫描子目录） |
| `qclaw` | QClaw | `~/.qclaw/skills` | - |
| `easyclaw` | EasyClaw | `~/.easyclaw/skills` | - |
| `autoclaw` | AutoClaw | `~/.openclaw-autoclaw/skills` | - |
| `workbuddy` | WorkBuddy | `~/.workbuddy/skills` | - |

其中 Hermes Agent 开启了 `recursive_scan: true`，会递归扫描 skills 目录下包含 SKILL.md 的子目录，适用于有嵌套分类目录结构的工具。

## 6. 预设（Presets）处理

个人助手 Agent 有意被排除在 coding 预设系统之外，拥有自己的 workspace（`app/src-tauri/src/commands/presets.rs`）。这意味着预设技能条（PresetBar）只对 coding 类 Agent 生效，个人助手类 Agent 不展示预设。

## 7. 自定义 Agent 支持

用户可以通过"自定义工具"功能添加自定义的个人助手类 Agent。`CustomToolDef` 结构体包含 `category` 字段，新建自定义 Agent 时可选择归类为 personal 或 coding。自定义 personal Agent 的持久化和加载逻辑在 `app/src-tauri/src/core/tool_service.rs`。

## 总结

「个人助手」是项目对 **通用个人助手类 Agent**（OpenClaw、Hermes Agent、WorkBuddy 等）的独立 UI 分组，与编程类 Agent 在功能上完全对等，只是产品定位不同（通用个人助手 vs 编程工具）。代码层面通过 `ToolCategory` 枚举做了轻量分类，复用 100% 的 workspace 交互逻辑——组件、路由、拖拽、技能管理、同步机制全部共享，仅文案和路由路径不同。"全部个人助手"是该分组下的概览页入口，对应路由 `/personal-workspace`。
