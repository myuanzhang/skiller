## 二、核心架构：Agent Skill 多目录系统

**术语对齐**：Agent = tool = `ToolAdapter`（代码里叫 tool/ToolAdapter，UI 与文档叫 Agent）。

### ToolAdapter 的三类目录
定义于 `app/src-tauri/src/core/tool_adapters.rs`。每个 adapter 现在有三类 skill 目录：

| 字段 | 含义 | 举例 | 可写？ |
|---|---|---|---|
| `relative_skills_dir` | 主目录 | `.codex/skills` | ✅ |
| `additional_scan_dirs` | 额外共享目录 | `.agents/skills` | ✅ |
| `readonly_scan_dirs` | 只读厂商/插件目录 | `.codex/plugins/cache` | ❌ 只读 |

关键方法：
- `all_scan_dirs()` → 主目录 + 已存在的额外目录（去重）。
- `readonly_existing_scan_dirs()` → 已存在的只读目录。

### 两条独立的 skill 读取路径（改动时别只改一条）
> 「全局工作区只扫主目录」正是最初的 bug 根源——发现流已支持多目录，工作区没跟上。

1. **发现流**：`core/scanner.rs`（Install 页，扫描本地未纳管 skill）。
2. **全局工作区**：`commands/agent_workspace.rs::read_agent_local_skills`（点某个 Agent 看它的 skill 列表）。现在会遍历全部三类目录、按 canonical path 去重、并给只读目录的 skill 打 `read_only` 标记。

### 前后端数据模型
- 后端 `ToolInfo`（`core/tool_service.rs`）→ `ToolInfoDto`（`commands/tools.rs`）暴露给前端：
  - `skills_dir`（主目录）、`scan_dirs`（主+额外）、`readonly_dirs`（只读）。
- 后端 `ProjectSkillInfo`（`core/project_scanner.rs`）→ 前端 `ProjectSkill`（`app/src/lib/tauri.ts`）：
  - 含 `path`（绝对路径）、`relative_path`、以及 `read_only`（只读标记）。
- 前端主界面：`app/src/views/WorkspaceView.tsx`（Agent 页标题栏 + skill 卡片）。

## 三、开发新 Agent/Skill 功能时的关键提示（踩过的坑）

1. **给某 Agent 加目录**：改 `default_tool_adapters()` 里对应 adapter 的 `additional_scan_dirs`（可写）或 `readonly_scan_dirs`（只读）。目前 **Codex / Claude Code / Grok** 配了 `readonly_scan_dirs`（分别是 `.codex/plugins/cache`、`.claude/plugins/cache`、`.grok/bundled/skills` + `.grok/marketplace-cache`）；其它 Agent 的插件/内置目录尚未调研配置。

2. **给结构体加字段 = 波及全部字面量**：`ToolAdapter` 有约 56 处字面量（`tool_adapters.rs` + `scanner.rs` + `agent_workspace.rs` 测试），`ProjectSkillInfo` 有多处（`project_scanner.rs` + `projects.rs` + 测试）。加字段必须同步更新**所有**构造点，否则编译不过。这也是为什么多目录那批改动合成了一个 commit（中间任何拆分都无法独立编译）。

3. **写操作必须过后端只读守卫**：任何新的 skill 写操作（import/update/delete 类）都要在后端调 `ensure_not_read_only`（`agent_workspace.rs`）拒绝只读 skill——不能只靠前端隐藏按钮。防御要在后端。

4. **只读目录要深扫**：只读目录（插件缓存）用 `scanner::collect_skill_dirs`，它能挖 `<pkg>/<version>/skills/<skill>` 这种嵌套 bundle 布局。**不要**用 `read_linked_workspace_skills`——它的 `should_skip_dir` 会故意跳过 `<x>/skills/` bundle 布局。参考现成的 `project_scanner::read_readonly_skills`。

5. **前端打开本地目录/文件**：`opener:default` 权限**不含** `allow-open-path`，前端直接调 `@tauri-apps/plugin-opener` 的 `openPath` 会被 ACL 拒绝。要走**后端受校验命令**——参考 `open_agent_scan_dir`（校验路径属于该 agent 的扫描目录）、`open_skill_local_copy`、`open_skill_source`。

6. **打开外部链接**（http/https）用前端 `openUrl` 即可（`opener:default` 允许），但**打开本地路径**必须走后端。

7. **i18n 三语齐全**：新增文案要同时加到 `en.json` / `zh.json` / `zh-TW.json`，且提交前 `python3 -c "import json; json.load(open(...))"` 校验合法。
